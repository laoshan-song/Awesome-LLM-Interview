# LoRA 及参数高效微调（PEFT）

## 面试高频考点
- LoRA 的原理是什么？为什么有效？
- rank 怎么选？
- LoRA 和全量微调的区别？

---

## 为什么需要 PEFT？

全量微调 70B 模型需要数百 GB 显存，成本极高。PEFT 只训练少量参数，保持大部分权重冻结。

---

## LoRA（Low-Rank Adaptation）

**核心思想**：预训练权重的更新量 ΔW 是低秩的，可以分解为两个小矩阵：

```
W' = W + ΔW = W + A * B
其中 A ∈ R^(d×r), B ∈ R^(r×k), r << min(d,k)
```

**训练过程：**
- W（预训练权重）冻结
- A 用随机高斯初始化，B 初始化为 0（保证初始 ΔW=0）
- 只训练 A 和 B

**参数量对比（d=k=4096, r=16）：**
- 全量：4096 × 4096 = 16.7M 参数
- LoRA：4096×16 + 16×4096 = 131K 参数（节省 99.2%）

---

## Rank 的选择

| rank | 适用场景 |
|------|----------|
| 4~8 | 资源极有限，简单任务 |
| 16~32 | 常见选择，平衡效果与效率 |
| 64~128 | 复杂任务，接近全量微调效果 |

---

## QLoRA

在 LoRA 基础上将基础模型量化为 4bit（NF4 格式）：
- 65B 模型可在单张 48GB GPU 上微调
- 性能接近 16bit 全量微调

---

## 其他 PEFT 方法对比

| 方法 | 核心思路 | 特点 |
|------|----------|------|
| LoRA | 低秩分解权重更新 | 推理可零开销合并 |
| QLoRA | LoRA + 4bit 量化 | 极低显存 |
| Prefix Tuning | 在 K、V 前拼接可训练向量 | 不修改模型权重 |
| Adapter | 在每层插入小型 MLP | 推理有额外延迟 |

---

## 面试延伸

**Q：LoRA 为什么初始化 B=0？**
> 保证训练开始时 ΔW=A·B=0，即 LoRA 层的初始输出与原始模型完全相同，不引入随机噪声破坏预训练知识。

**Q：LoRA 推理时有额外开销吗？**
> 可以将 LoRA 权重合并回原始模型（W' = W + AB），推理时零开销。也可以保持分离，支持动态切换不同的 LoRA 适配器。
