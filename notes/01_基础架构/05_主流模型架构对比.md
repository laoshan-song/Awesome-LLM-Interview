## LLaMA 系列的关键改进

相比原始 Transformer，LLaMA 做了以下四个关键改动：

### 1. Pre-RMSNorm（前置归一化）
将 LayerNorm 移到残差块前面（Pre-Norm），并使用 RMSNorm 替代 LayerNorm。

**RMSNorm vs LayerNorm：**
```
LayerNorm: y = (x - mean) / sqrt(var + ε) × γ + β
RMSNorm:   y = x / RMS(x) × γ    其中 RMS(x) = sqrt(mean(x²))
```
- RMSNorm 不减均值，计算量约少 7%
- 实验表明性能相当，现代 LLM（LLaMA、Qwen、DeepSeek）普遍采用

### 2. SwiGLU 激活函数
```
FFN(x) = (xW₁ ⊙ σ(xW₁)) · xW₃    （Swish Gate）
```
- 相比 ReLU 引入了门控机制（GLU），提供更平滑的梯度
- 需要 3 个矩阵（而非 2 个），FFN 隐藏层维度通常设为 2/3 × 4d 以保持参数量不变

### 3. RoPE 位置编码
见 [04_位置编码.md](../01_基础架构/04_位置编码.md)

### 4. GQA（分组查询注意力）
LLaMA 2 70B 和 LLaMA 3 全系列采用，KV Cache 显存减少 N/G 倍（G 为组数）。

## DeepSeek-V2 的 MLA（多头潜在注意力）

**问题**：标准 MHA 的 KV Cache 随序列长度线性增长，128K 上下文时极大。

**MLA 方案**：将 K、V 压缩到一个低维潜在向量（Latent Vector）：
```
KV Cache 只需存储低维潜在向量 c_KV ∈ R^(d_c)，而非完整 K、V
推理时从 c_KV 解压还原 K、V
```

**效果**：KV Cache 相比 MHA 减少 **93.3%**，同时性能优于 GQA。

## Decoder-only 为何成为主流？

| 维度 | Encoder-only | Encoder-Decoder | Decoder-only |
|------|-------------|-----------------|--------------|
| 代表 | BERT | T5、BART | GPT、LLaMA |
| 预训练目标 | MLM（掩码预测）| seq2seq | 下一个 Token 预测 |
| 生成能力 | 弱 | 强 | 强 |
| Scaling 表现 | 一般 | 一般 | **最好** |
| 适用场景 | 分类、NER | 翻译、摘要 | 通用生成 |

Decoder-only 在 Scaling Law 下表现最优，且统一了预训练和生成任务，是目前 LLM 的主流选择。

## 面试延伸

**Q：为什么 LLaMA 3 把词表从 32K 扩到 128K？**
> 更大词表对多语言和代码支持更好，单个 Token 覆盖更多字符，序列长度更短，推理效率更高。代价是 Embedding 层参数量增加。

**Q：SwiGLU 为什么要用 2/3 × 4d 而不是 4d 作为隐藏层维度？**
> SwiGLU 使用了 3 个权重矩阵（W₁, W₂, W₃），如果用标准 4d 维度参数量会比原始 FFN 多 50%。调整为 2/3 × 4d 可以保持总参数量大致相同。

**Q：DeepSeek-V2 的 MoE 和 MLA 分别解决什么问题？**
> MoE 解决了"如何用更少计算量扩大模型容量"的问题（236B 参数但每次只激活 21B）；MLA 解决了长上下文下 KV Cache 显存爆炸的问题（减少 93% KV Cache）。两者都是推理效率的优化。
