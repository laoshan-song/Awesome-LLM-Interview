## 一、DeepSeek 系列

DeepSeek（深度求索）是 2025-2026 年最受全球关注的中国 AI 公司，以极低训练成本实现顶级性能著称。

### 模型谱系

| 模型 | 发布时间 | 参数 | 类型 | 核心亮点 |
|------|---------|------|------|---------|
| DeepSeek-V2 | 2024.05 | 236B (21B激活) | MoE 基础模型 | MLA 首次提出，KV Cache 减少 93% |
| DeepSeek-V3 | 2024.12 | 671B (37B激活) | MoE 基础模型 | FP8训练、无辅助Loss负载均衡、MTP多Token预测 |
| DeepSeek-R1 | 2025.01 | 671B | 推理模型 | GRPO纯RL训练，Long CoT涌现，$550万训练成本 |
| DeepSeek-V4 | 2026 | 待确认 | MoE 基础模型 | 架构持续演进（详情以官方发布为准）|

### DeepSeek-V3 关键技术

**① 无辅助损失负载均衡（Auxiliary-Loss-Free Load Balancing）**

传统 MoE 用辅助 Loss 强制均衡专家使用率，但会干扰主任务 Loss。DeepSeek-V3 改为动态调整每个专家的偏置项（bias），将负载均衡从 Loss 层面移到路由层面，不影响模型优化目标。

**② FP8 混合精度训练**

首次在生产规模（671B 参数）使用 FP8 进行前向和反向传播，显著降低显存占用和通信开销，H800 集群训练成本约 558 万美元。

**③ MTP（Multi-Token Prediction）**

除主 token 预测外，额外预测未来 N 个 token，提升训练效率和模型对长程依赖的建模能力，推理时也可用于投机解码加速。

**④ MLA（Multi-head Latent Attention）**

继承自 V2，将 K/V 压缩到低维潜在向量 c_KV，KV Cache 相比 MHA 减少 **93.3%**，是支撑 128K 长上下文的关键。

```
标准 MHA KV Cache：每层存储 2 × num_heads × head_dim × seq_len
MLA KV Cache：每层只存储 d_c（远小于 num_heads × head_dim）× seq_len
推理时从 c_KV 解压还原完整 K/V
```

### DeepSeek-R1 训练流程

```
阶段1：冷启动 SFT（少量 Long CoT 数据）
  ↓
阶段2：GRPO 纯 RL（规则奖励：数学验证 + 格式检查）
  ↓  自发涌现：自我验证、回溯、反思
阶段3：拒绝采样 SFT（RL 生成的高质量轨迹蒸馏）
  ↓
阶段4：RL 精调（加入通用能力对齐）
```

**核心论文**：[arXiv:2412.19437](https://arxiv.org/abs/2412.19437)（V3）、[arXiv:2501.12948](https://arxiv.org/abs/2501.12948)（R1）

## 二、Qwen 系列（阿里巴巴）

通义千问（Qwen）是阿里巴巴达摩院出品，目前已迭代至 **Qwen3**（2025年4月发布），是开源生态中综合能力最强的模型系列之一。

### 模型谱系

| 模型 | 参数 | 上下文 | 核心特性 |
|------|------|--------|---------|
| Qwen2 7B/72B | 7B/72B | 128K | GQA、SwiGLU、151K词表 |
| Qwen2.5 7B~72B | 多规格 | 128K | 强化代码/数学能力，指令遵循提升 |
| **Qwen3 Dense** | 0.6B~32B | 128K | 混合思考模式，119语言 |
| **Qwen3-235B-A22B** | 235B (22B激活) | 128K | MoE 旗舰，对标 GPT-4o |

### Qwen3 核心创新：混合思考模式（Hybrid Thinking）

Qwen3 在同一模型中同时支持两种模式，通过系统提示切换：

```python
# 思考模式（慢，精准）——复杂推理/数学/代码
messages = [{"role": "system", "content": "/think"}]

# 非思考模式（快，流式）——日常对话/翻译
messages = [{"role": "system", "content": "/no_think"}]
```

**与 o1 的本质区别**：o1 的思维链对用户不可见（黑盒）；Qwen3 的 `<think>` 标签内容用户可见，且可以灵活切换是否激活推理模式。

**训练数据**：~36 万亿 tokens，覆盖 119 种语言和方言。

**核心论文**：[Qwen3 技术报告（2025）](https://arxiv.org/abs/2505.09388)

## 三、GLM 系列（清华大学 × 智谱AI）

ChatGLM 由清华大学 KEG 实验室与智谱AI联合研发，是最早开源的中文对话大模型之一，在中文学术/企业界有广泛使用。

### 模型谱系

| 模型 | 参数 | 上下文 | 特点 |
|------|------|--------|------|
| ChatGLM-6B | 6B | 2K | 首个开源中文对话模型（2023） |
| ChatGLM2-6B | 6B | 32K | 引入 RoPE + FlashAttention |
| ChatGLM3-6B | 6B | 128K | 工具调用、代码解释器 |
| **GLM-4** | 未公开 | 128K | 对标 GPT-4，多模态 GLM-4V |
| **GLM-4-Long** | 未公开 | **1M** | 超长上下文 |
| **GLM-4-AllTools** | 未公开 | 128K | 原生工具调用、网络搜索 |

### GLM 架构特点

相比标准 Transformer，GLM 系列有以下特点：

**① GLM 预训练目标（早期）**：自回归填空（Autoregressive Blank Infilling），同时具备 MLM 和 Causal LM 能力（现已转向标准 Causal LM）

**② 双向注意力 + 单向注意力混合**（GLM-130B 时代）：部分 token 使用双向注意力，后来统一改为 Decoder-only

**③ RoPE 位置编码 + Pre-RMSNorm**：GLM-4 系列与 LLaMA 架构对齐，采用主流设计

**核心仓库**：[github.com/THUDM/ChatGLM3](https://github.com/THUDM/ChatGLM3)、[github.com/THUDM/GLM-4](https://github.com/THUDM/GLM-4)

## 四、MiniMax（名之又名）

MiniMax 是上海的 AI 独角兽公司（非 minimax 博弈），以**超长上下文 + MoE** 组合见长。

### 模型谱系

| 模型 | 发布时间 | 参数 | 上下文 | 亮点 |
|------|---------|------|--------|------|
| MiniMax-Text-01 | 2025.01 | 456B (45.9B激活) | **1M tokens** | Lightning Attention，开源 |
| MiniMax-M1 | 2025.05 | 456B (45.9B激活) | 1M tokens | 推理模型，对标 DeepSeek-R1 |

### MiniMax 核心创新：Lightning Attention

**问题**：标准 Softmax Attention 的 KV 矩阵随序列长度 O(n²) 增长，1M token 场景根本无法使用。

**Lightning Attention**（线性注意力变体）：

```
标准 Attention:   Softmax(QK^T / √d) · V    —— O(n²) 复杂度
Linear Attention: φ(Q) · (φ(K)^T · V)       —— O(n) 复杂度
                  通过核函数近似替代 softmax
```

MiniMax-Text-01 采用**混合注意力**：大部分层使用 Lightning Attention（线性复杂度处理长程依赖），间隔插入标准 Softmax Attention 层（保持局部精度），在 1M 上下文下实现高效推理。

**核心论文**：[arXiv:2501.08313](https://arxiv.org/abs/2501.08313)

## 面试延伸

**Q：DeepSeek 为什么能用 $558 万训练出媲美 GPT-4 的模型？**
> 三个核心因素：① FP8 精度训练大幅降低显存和通信开销；② MoE 稀疏激活（671B 总参数但每次只用 37B），训练 FLOPs 与同等能力的 Dense 模型相比节省数倍；③ 工程极度优化（跨节点 EP 通信与计算重叠、流水线气泡最小化）。同时中国工程师的人力成本也显著低于 OpenAI。

**Q：Qwen3 的混合思考模式在工程上如何实现？**
> 本质是在 SFT 和 RLVR 阶段用两类数据混合训练：带 `<think>...</think>` 标签的长推理数据（激活思考模式）和不带标签的直接回复数据（非思考模式）。推理时通过系统提示中的 `/think` 或 `/no_think` 控制模型行为分支，实现方式类似"软开关"，而非两个独立模型。

**Q：为什么 MiniMax 的 1M 上下文在工程上可行，而普通 Transformer 不行？**
> 标准 Softmax Attention 的注意力矩阵是 n×n 的，1M token 下 n²=10¹² 完全无法放入 GPU 显存。MiniMax 的 Lightning Attention（线性注意力）通过核函数近似将复杂度降到 O(n)，代价是局部精度略低于 Softmax。混合架构（线性层 + 少量 Softmax 层）在保持效率的同时补偿了精度损失。
