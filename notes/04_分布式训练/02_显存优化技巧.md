## 显存占用公式

混合精度训练（AdamW 优化器），参数量 Φ：

| 组成部分 | 精度 | 占用 |
|----------|------|------|
| 模型参数（前向用）| FP16/BF16 | 2Φ bytes |
| 梯度 | FP16/BF16 | 2Φ bytes |
| 参数主副本（优化器用）| FP32 | 4Φ bytes |
| Adam 一阶矩（m）| FP32 | 4Φ bytes |
| Adam 二阶矩（v）| FP32 | 4Φ bytes |
| **合计** | | **16Φ bytes** |

**LLaMA-7B 示例**（Φ = 7B）：
```
16 × 7 × 10⁹ bytes ≈ 112 GB
```
单张 80GB A100 放不下，需要并行或优化。

激活值（Activations）通常与 batch size × seq_len 成正比，是另一个显存大户。

---

## 混合精度训练（AMP）

**核心思路**：前向和反向用低精度（FP16/BF16）加速，优化器更新用 FP32 保持精度。

**FP16 vs BF16：**

| 特性 | FP16 | BF16 |
|------|------|------|
| 指数位 | 5 位 | 8 位（与 FP32 相同）|
| 尾数位 | 10 位 | 7 位 |
| 动态范围 | 小，易溢出 | 大，与 FP32 相当 |
| 精度 | 高 | 低 |
| 是否需要 Loss Scaling | **是** | 否 |
| 适用硬件 | V100、T4 | A100、H100 |

**Loss Scaling**：FP16 动态范围小，梯度容易下溢为 0。通过将 loss 乘以一个大数（scale factor）再反向，最后更新前除回来，避免梯度消失。

---

## 梯度检查点（Gradient Checkpointing）

**问题**：标准反向传播需要保存所有层的激活值用于计算梯度，激活值显存随层数线性增长。

**方案**：只保存部分层的激活值（checkpoint），其余层在反向传播时重新计算。

```
显存节省：O(√N) 数量级（N 为层数）
计算代价：约增加 30% 的计算量（重算一次前向）
```

**PyTorch 使用：**
```python
from torch.utils.checkpoint import checkpoint
output = checkpoint(layer, input)
```

**Selective Recomputation**：只对计算便宜但显存大的操作重算（如 Attention），对计算昂贵的操作（如 FFN）仍然保存。

---

## 梯度累积（Gradient Accumulation）

当显存不足以放下大 batch 时，将多个小 batch 的梯度累加后再更新：

```python
for i, (input, label) in enumerate(dataloader):
    loss = model(input, label) / accumulation_steps
    loss.backward()
    if (i + 1) % accumulation_steps == 0:
        optimizer.step()
        optimizer.zero_grad()
```

**效果**：`effective_batch_size = batch_size × accumulation_steps`，显存占用不变。
**注意**：使用 DDP 时需配合 `no_sync()` 避免每个 micro-step 都触发 AllReduce。

## 优化器显存优化

**8-bit Adam（bitsandbytes）**：
- 将优化器状态从 FP32 量化为 INT8
- 显存节省约 75%（优化器状态部分）
- 精度损失极小

**Paged Optimizer**：
- 优化器状态分页到 CPU 内存，GPU 显存不够时自动换出
- QLoRA 使用此方案，65B 模型可在 48GB GPU 上微调

---

## Flash Attention 节省显存

标准 Attention 需要存储 N×N 的注意力矩阵：O(N²) 显存。
Flash Attention 通过分块计算，显存降至 O(N)，对长序列效果显著。

---

## 综合显存节省效果

| 技术 | 显存节省 | 计算代价 |
|------|---------|---------|
| BF16 混合精度 | ~50%（vs FP32）| 无 |
| 梯度检查点 | ~60-70% 激活值 | +30% 计算 |
| ZeRO-3 | ~1/N（N 卡）| 通信增加 |
| 8-bit Adam | ~75% 优化器状态 | 无 |
| Flash Attention | O(N²)→O(N) | 无（更快）|

## 面试延伸

**Q：为什么 Adam 比 SGD 多占 3 倍显存？**
> SGD 只需存储梯度（2Φ bytes），Adam 额外需要一阶矩 m 和二阶矩 v（各 4Φ bytes，FP32），合计多 8Φ bytes。

**Q：梯度检查点一定选每隔一层保存吗？**
> 不一定。理论上每隔 √N 层保存一次可以把激活值显存从 O(N) 降到 O(√N)。实践中通常每个 Transformer Block 保存一次，权衡实现复杂度和效果。

**Q：显存不够训练时，优先用哪个优化手段？**
> 优先级：混合精度（无损）→ 梯度累积（无损）→ 梯度检查点（轻微计算代价）→ ZeRO（需要多卡）→ 8-bit Adam → CPU Offload（最后手段，速度慢）。
