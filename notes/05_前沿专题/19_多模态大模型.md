## 技术路线对比

2025 年多模态 LLM 已从"给 LLM 加视觉插件"进化为**原生 Any-to-Any 架构**。主要有两条路线：

### 路线一：连接器桥接（Connector-based）

冻结预训练视觉编码器，通过 Projector 将视觉特征对齐到 LLM 的 Token 空间：

```
图像 → Vision Encoder（CLIP/SigLIP）→ 视觉特征
                                         ↓
                                 Projector（MLP / Q-Former）
                                         ↓
                               与文本 Token 拼接 → LLM
```

代表模型：LLaVA（MLP Projector）、InternVL（动态分辨率）、Qwen-VL、mPLUG-Owl

**优点**：视觉和语言模块可独立升级；训练成本低（可冻结大部分参数）

### 路线二：早期融合（Early Fusion / Omni Architecture）

视觉 Token 与文本 Token 在同一个 Transformer 中联合建模，无显式分离：

```
图像 → 视觉 Tokenizer（离散/连续）→ Image Tokens
文本 → 文本 Tokenizer              → Text Tokens
                  ↓
           统一 Transformer（全模态联合预训练）
                  ↓
         文本输出 / 图像输出 / 语音输出
```

代表模型：GPT-4o、Gemini 2.0、LLaMA 3.2 Vision、Chameleon

**优点**：更自然的多模态交互；支持任意模态的输出（Any-to-Any）

## 视觉编码器对比

| 编码器 | 预训练方式 | 优点 | 代表使用 |
|--------|----------|------|---------|
| CLIP ViT | 图文对比学习 | 强语义对齐，多语言支持 | LLaVA、BLIP-2 |
| SigLIP | Sigmoid 损失（无 softmax）| 支持任意 batch size，性能更强 | InternVL、PaliGemma |
| DINOv2 | 自监督（无文本）| 空间特征更丰富，适合目标检测 | - |

## 高分辨率图像处理

**问题**：CLIP ViT 通常在 224×224 或 336×336 分辨率训练，直接处理高分辨率图像效果差。

### 动态分辨率（Dynamic Resolution）— InternVL 方案

```
高分辨率图像（如 1024×768）
         ↓
    自适应切片（Adaptive Tiling）
    ┌────┬────┬────┐
    │Tile│Tile│Tile│
    ├────┼────┼────┤   每个 Tile = 448×448
    │Tile│Tile│Tile│
    └────┴────┴────┘
         ↓
    每个 Tile 独立经过 ViT 编码
         ↓
    拼接 + 缩略图 Token（全局信息）
         ↓
    送入 LLM
```

**效果**：InternVL 2.5 在文档理解、OCR、图表理解等高分辨率任务上显著优于固定分辨率方案。

## Q-Former（BLIP-2 架构）

Q-Former 是另一种连接方式，使用可学习的查询向量通过交叉注意力提取视觉特征：

```
固定查询向量 Q（32 个）
        ↓
Cross-Attention ← 图像特征（视觉编码器输出）
        ↓
   32 个视觉 Token（固定数量，与图像分辨率无关）
        ↓
        LLM
```

**优点**：大幅压缩视觉 Token 数量，降低 LLM 的上下文负担  
**缺点**：固定 32 Token 可能丢失细粒度信息，不适合高分辨率场景

## 多模态对齐训练阶段

通常分三阶段（以 LLaVA 为例）：

| 阶段 | 训练数据 | 冻结参数 | 目标 |
|------|---------|---------|------|
| Stage 1：特征对齐 | 图文对（概念描述）| LLM + Vision Encoder | 训练 Projector 对齐语义空间 |
| Stage 2：指令微调 | 视觉指令数据 | Vision Encoder | 联合训练 Projector + LLM |
| Stage 3（可选）：RLHF | 人类偏好数据 | - | 提升对话质量和安全性 |

## 视频理解的额外挑战

- **时序建模**：帧之间的时间依赖关系（不是 N 张独立图片）
- **帧采样策略**：均匀采样 vs. 关键帧检测 vs. 密集采样
- **Token 爆炸**：1 分钟 1fps = 60 帧，每帧 256 Token = 15360 Token，超出上下文限制
- **解决方案**：时序压缩（将多帧 Token 合并）、稀疏注意力

## 面试延伸

**Q：为什么视觉 Token 数量很重要？**
> 每张图片通常生成 256-4096 个 Token，大大增加上下文长度，推理延迟和显存随之增加。Token 压缩（如 Q-Former）和稀疏注意力是重要优化方向。

**Q：多模态模型为什么在 OCR 和图表理解上困难？**
> 文字和图表需要高分辨率感知（细节像素级别），标准 ViT 的感受野设计偏向语义特征而非高频细节，动态分辨率切片 + 更高分辨率 ViT 是主要解决路径。

**Q：语音模态通常如何加入？**
> 语音经过 Whisper 等 ASR 编码器提取声学特征后，通过类似视觉 Projector 的桥接结构注入 LLM。原生语音模型（如 GPT-4o 语音模式）则在声音 Token 层面直接与文本 Token 联合建模。
