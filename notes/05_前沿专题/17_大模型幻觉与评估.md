# 大模型幻觉与评估

## 面试高频考点
- 什么是幻觉（Hallucination）？如何分类？
- 幻觉产生的根本原因是什么？
- 如何缓解幻觉？RAG 能完全解决吗？
- MMLU、HumanEval、GSM8K 分别评测什么？
- LLM-as-Judge 有哪些偏差？

---

## 幻觉的定义与分类

**幻觉**：模型生成的内容在事实上不正确、与输入矛盾、或无法被现实世界知识验证的现象。

### 分类框架

| 类型 | 描述 | 示例 |
|------|------|------|
| **事实性幻觉** | 与客观事实不符 | "爱因斯坦 1921 年因相对论获诺贝尔奖"（实为光电效应）|
| **忠实性幻觉** | 与输入文本/上下文矛盾 | 摘要中出现原文没有的信息 |
| **推理性幻觉** | 推理链正确但结论错误 | 数学题中间步骤对但最终答案算错 |

---

## 幻觉产生的原因

**数据层面：**
- 训练数据中存在矛盾、过时、错误信息
- 长尾知识（罕见事实）训练样本不足
- 数据截止日期导致知识过时

**训练层面：**
- 语言模型的目标是预测下一个 token，而非"说真话"
- RLHF 中 Sycophancy（拍马屁效应）：模型学会说用户想听的话而非真实答案
- Reward Hacking：奖励模型被欺骗，高分但不正确

**推理层面：**
- 自回归生成的滚雪球效应：早期错误影响后续所有输出
- 注意力机制对长文本中关键信息的权重不足
- 高 Temperature 采样增加随机性，引入错误

---

## 缓解幻觉的方法

### 1. RAG（检索增强生成）
将外部知识实时注入上下文，让模型基于检索到的事实回答，而非依赖参数记忆。

**局限性**：
- 检索质量差时反而引入噪声
- 模型可能忽略检索内容，仍凭记忆回答
- 无法解决推理性幻觉

### 2. Self-Refine（自我修正）
三角色迭代框架：
```
生成 → 批评（Critique）→ 修正（Refine）→ 再批评 → ...
```
让模型对自己的输出进行评估和修改，迭代提升质量。

### 3. CoVe（Chain of Verification）
Meta AI 提出，四步独立验证：
1. 生成初始回答
2. 制定一系列验证问题（"爱因斯坦的诺贝尔奖是因为什么？"）
3. 独立回答每个验证问题（避免受初始答案影响）
4. 基于验证结果修正最终答案

### 4. RLHF 与 DPO
通过人类反馈训练模型倾向于给出真实、有依据的回答，但 Sycophancy 问题难以完全消除。

### 5. 不确定性表达
训练模型在不确定时主动说"我不确定"，而非编造答案。

## 主流评测基准

### MMLU（大规模多任务语言理解）
- **内容**：57 个学科的选择题（数学、法律、医学、物理等）
- **格式**：四选一选择题
- **意义**：综合知识评估，最常用的 benchmark
- **当前状态**：GPT-4 约 86%，接近饱和

### HumanEval
- **内容**：164 道 Python 编程题，给函数签名和文档字符串，生成函数体
- **指标**：Pass@k（k 次采样中至少一次通过测试的概率）
- **意义**：代码生成能力评估

### GSM8K
- **内容**：8500 道小学数学应用题，需要多步推理
- **意义**：数学推理能力，CoT 效果显著的典型 benchmark
- **当前状态**：顶级模型已接近 100%

### MT-Bench
- **内容**：80 道多轮对话题（8 类任务：推理、数学、写作、编程等）
- **评测方式**：GPT-4 打分（1-10 分）
- **意义**：对话质量和指令遵循综合评估

### AlpacaEval 2.0
- **内容**：805 个 instruction，与 GPT-4-Turbo 对比胜率
- **评测方式**：LLM-as-Judge，计算 Length-Controlled Win Rate
- **意义**：开放式生成质量对比

---

## LLM-as-Judge

**原理**：用更强的 LLM（如 GPT-4）作为评估者，对模型输出打分或进行 A/B 比较。

**优点：**
- 相关性高，接近人类判断
- 成本低于人工评估
- 可以大规模自动化

**已知偏差：**

| 偏差类型 | 描述 |
|---------|------|
| 位置偏差 | 倾向于选择第一个或第二个选项 |
| 冗长偏差 | 倾向于打分更长的回复更高 |
| 自我增强偏差 | GPT-4 倾向于打高分给 GPT-4 生成的内容 |
| 格式偏差 | Markdown 格式的回复得分更高 |
| Sycophancy | 被用户声明影响（"我认为答案是A"）|

**缓解方法：**
- 交换回复顺序取平均（消除位置偏差）
- 引入 Length-Controlled 指标（消除冗长偏差）
- 使用多个不同 LLM 评估取多数

---

## 2024-2025 前沿

**基准饱和危机**：MMLU、GSM8K 等经典 benchmark 已被顶级模型接近满分，区分度丧失。
**新方向**：LiveBench（防污染实时更新）、GPQA（博士级科学题）、SimpleQA（事实性简单问答）。

**FActScore**：细粒度事实核查，将生成文本拆分为原子事实，逐一检索验证，比整体评分更精确。

## 面试延伸

**Q：RAG 能完全解决幻觉问题吗？**
> 不能。RAG 能缓解知识截止和事实性幻觉，但对推理性幻觉（模型推理过程的错误）无效。即使提供了正确的上下文，模型仍可能忽略或误用这些信息。完全消除幻觉目前没有解决方案。

**Q：为什么 LLM 在不知道的问题上不说"不知道"？**
> 语言模型的训练目标是最大化下一个 token 的概率，而不是判断自己是否知道答案。模型没有"元认知"能力，它只是在续写概率最高的内容。通过 RLHF 可以部分训练出"不确定时说不知道"的行为，但不彻底。

**Q：评测大模型时为什么不能只看 MMLU？**
> MMLU 是知识选择题，无法评估生成质量、代码能力、对话连贯性、指令遵循等重要维度。不同任务所需能力不同，需要多个 benchmark 综合评估，且要注意测试集污染（模型可能见过这些题目）。
