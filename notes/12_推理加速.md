# 推理加速

## 面试高频考点
- Flash Attention 的原理？
- 连续批处理（Continuous Batching）是什么？
- 投机采样如何加速推理？

---

## Flash Attention

**问题**：标准 Attention 需要将 n×n 的注意力矩阵写入 HBM（GPU 显存），IO 是瓶颈，不是计算。

**解决**：分块（Tiling）计算，将中间结果留在 SRAM（片上缓存），减少 HBM 读写次数。

**效果：**
- 内存使用从 O(n²) 降至 O(n)
- 训练速度提升 2~4x，数学上等价于标准 Attention（无精度损失）

---

## 连续批处理（Continuous Batching）

**传统批处理问题**：一批请求必须等最长的那条生成完才能释放，GPU 等待时间长。

**Continuous Batching**：请求完成后立即将新请求插入批次，无需等待整批完成。

**效果**：吞吐量提升数倍，GPU 利用率大幅提高。

---

## vLLM

集成了 PagedAttention + Continuous Batching 的高性能推理框架：

| 特性 | 效果 |
|------|------|
| PagedAttention | 显存利用率 90%+，支持更大 batch |
| Continuous Batching | GPU 利用率大幅提升 |
| Prefix Caching | 相同 System Prompt 的 KV 跨请求共享 |
| 量化支持 | AWQ、GPTQ、FP8 |

---

## 投机采样（Speculative Decoding）

**核心思想**：小模型（Draft）快速生成多个候选 Token，大模型（Target）并行验证：
- 接受正确的 Token，拒绝并重采样错误的
- 输出分布与原始大模型完全一致（无精度损失）

**效果**：吞吐量提升 2~3x，收益在输出高度可预测时（代码补全）最大。

## 其他优化技术

| 技术 | 思路 |
|------|------|
| Tensor Parallelism | 将权重矩阵切分到多卡 |
| Pipeline Parallelism | 不同层分配到不同卡，流水线执行 |
| CUDA Graph | 固化计算图，减少 CPU-GPU 通信开销 |
| Chunked Prefill | 将长 Prefill 拆成多个 chunk 与 Decode 交错 |

---

## 面试延伸

**Q：Prefill 和 Decode 的瓶颈不同，怎么处理？**
> Prefill 是 compute-bound，可用更大 batch 提高利用率；Decode 是 memory-bound，关键是减少显存读写（量化、GQA）。Disaggregated Prefill/Decode（分离部署）是最新方向。
