# LLM 核心名词解释

> 创建日期：2026-05-28

---

## 基础概念

### Token
模型处理文本的最小单位，可以是一个词、子词或字符。例如 "unhappiness" 可能被切分为 ["un", "happiness"]。

### Tokenization（分词）
将原始文本转换为 Token 序列的过程。常见算法：BPE、WordPiece、SentencePiece。

### Embedding（词嵌入）
将 Token 映射为稠密向量的过程。向量空间中语义相近的词距离也相近。

### Vocabulary（词表）
模型能识别的所有 Token 的集合。GPT-2 词表大小约 50K，LLaMA 约 32K。

## 模型架构

### Transformer
2017 年 Google 提出的序列建模架构，核心是自注意力机制（Self-Attention），完全取代了 RNN。

### Self-Attention（自注意力）
序列中每个位置的 Token 与所有其他位置计算相关性，公式：
```
Attention(Q, K, V) = softmax(QK^T / sqrt(d_k)) * V
```

### Multi-Head Attention（多头注意力）
将 Q、K、V 分别投影到多个子空间并行做 Attention，然后拼接，增强模型捕获不同关系的能力。

### Q / K / V
Query（查询）、Key（键）、Value（值）。Q 与 K 计算注意力权重，再对 V 加权求和。

### KV Cache
推理时缓存已计算过的 K、V 矩阵，避免对历史 Token 重复计算，大幅加速自回归生成。

### Positional Encoding（位置编码）
Transformer 无序列顺序感知能力，通过位置编码注入位置信息。分为绝对位置编码（如 sinusoidal）和相对位置编码（如 RoPE、ALiBi）。

### RoPE（旋转位置编码）
通过旋转矩阵将位置信息编码进 Q、K，LLaMA / Qwen 等主流模型采用，支持外推到更长上下文。

### FFN（前馈网络）
Transformer 每层中 Attention 之后的两层全连接网络，负责非线性变换。

### Layer Norm（层归一化）
对每层的输出做归一化，稳定训练。Pre-Norm（归一化在残差之前）比 Post-Norm 训练更稳定。

## 训练相关

### Pre-training（预训练）
在大规模无监督语料上训练语言模型，目标通常是下一个 Token 预测（Causal LM）。

### Fine-tuning（微调）
在预训练模型基础上，用特定任务数据继续训练，调整模型参数。

### SFT（Supervised Fine-Tuning，有监督微调）
用人工标注的指令-回答对对预训练模型进行微调，使其学会遵循指令。

### RLHF（基于人类反馈的强化学习）
通过人类对模型输出的偏好标注训练奖励模型，再用 PPO 等算法优化语言模型，使其输出更符合人类期望。

### PPO（近端策略优化）
RLHF 中常用的强化学习算法，通过限制每次策略更新幅度来稳定训练。

### DPO（直接偏好优化）
RLHF 的简化替代方案，直接用偏好数据优化模型，无需单独训练奖励模型。

## 推理相关

### Autoregressive（自回归）
模型逐个生成 Token，每次生成依赖之前所有已生成的 Token。

### Greedy Decoding（贪心解码）
每步选择概率最高的 Token，速度快但多样性差。

### Beam Search（束搜索）
同时维护 K 条候选序列，综合考虑全局概率，质量优于贪心但计算量大。

### Temperature（温度）
控制输出分布的平滑程度。Temperature→0 趋近贪心，Temperature 越大输出越随机。

### Top-p / Top-k 采样
- **Top-k**：每步只从概率最高的 k 个 Token 中采样
- **Top-p（nucleus sampling）**：从累积概率超过 p 的最小 Token 集合中采样

### Perplexity（困惑度）
衡量语言模型好坏的指标，值越低表示模型对测试集预测越准确。

## 效率与部署

### Quantization（量化）
将模型权重从 FP32/FP16 压缩为 INT8/INT4 等低精度格式，减少显存占用和推理延迟。

### LoRA（低秩适配）
微调时只训练少量低秩矩阵而非全部参数，大幅减少训练显存和计算量。

### MoE（混合专家）
模型由多个"专家"子网络组成，每次推理只激活少数专家，在不增加推理成本的前提下扩大模型容量。

### Prefill / Decode
推理的两个阶段：Prefill 是对输入 Prompt 并行计算 KV Cache；Decode 是逐步自回归生成输出 Token。
