# 位置编码（Positional Encoding）

## 面试高频考点
- Transformer 为什么需要位置编码？
- RoPE 的原理是什么？为什么比绝对位置编码更好？
- ALiBi 如何实现长度外推？

---

## 为什么需要位置编码？

Self-Attention 的计算对输入顺序是**排列不变的**（permutation invariant）——打乱 Token 顺序，输出不变。语言有顺序，必须手动注入位置信息。

---

## 绝对位置编码（Sinusoidal）

原始 Transformer 论文使用，将位置 pos 编码为固定的正弦/余弦向量：

```
PE(pos, 2i)   = sin(pos / 10000^(2i/d_model))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))
```

**优点**：无需学习，可以外推到训练时未见过的长度。
**缺点**：位置信息以加法方式注入，与内容耦合弱，长距离外推效果差。

---

## 可学习的绝对位置编码

BERT、GPT-2 使用。直接将位置 ID 映射为可训练的 Embedding，与 Token Embedding 相加。

**缺点**：只能处理训练时见过的长度，无法外推。

---

## RoPE（旋转位置编码）

**来源**：LLaMA、Qwen、ChatGLM 等主流模型使用。

**核心思想**：不直接加位置向量，而是将位置信息通过**旋转矩阵**编码进 Q 和 K，使得 Q·K 的点积自然包含**相对位置**信息。

**数学直觉（2D 情形）：**
```
q_m = R(m) · q    # 将 query 旋转 m·θ 角度
k_n = R(n) · k    # 将 key 旋转 n·θ 角度
q_m · k_n = q · R(m-n) · k  # 点积只依赖相对距离 m-n
```

**优点：**
- 天然编码相对位置，无需显式计算相对距离
- 对长上下文外推友好（配合 YaRN、RoPE scaling 等技术）
- 实现高效，可与 Flash Attention 结合

**RoPE 长度外推方案：**
| 方法 | 思路 |
|------|------|
| Position Interpolation | 线性插值压缩位置 ID |
| YaRN | 分频率段差异化插值 |
| LongRoPE | 非均匀插值 + 微调 |

---

## ALiBi（Attention with Linear Biases）

**来源**：MPT 模型使用。

**核心思想**：不修改位置 Embedding，而是在注意力分数上直接加一个与距离成正比的**负偏置**：

```
score(i, j) = q_i · k_j / √d_k - m · |i - j|
```

其中 m 是每个注意力头的斜率（不同头不同值）。

**优点：**
- 天然惩罚远距离依赖，长文本外推效果好
- 无需修改已有模型结构

**缺点：**
- 相比 RoPE，在需要精确位置感知的任务（如代码）上表现稍弱

---

## 对比总结

| 方法 | 类型 | 外推能力 | 代表模型 |
|------|------|----------|----------|
| Sinusoidal | 绝对 | 一般 | 原始 Transformer |
| Learned | 绝对 | 差（无法外推）| BERT、GPT-2 |
| RoPE | 相对 | 好（可扩展）| LLaMA、Qwen |
| ALiBi | 相对偏置 | 好 | MPT、BLOOM |

---

## 面试延伸

**Q：RoPE 为什么比加法位置编码更适合外推？**
> 加法编码的位置向量在训练范围外没有见过，直接外推会有分布偏移。RoPE 编码的是相对距离，长文本中新出现的相对距离可以通过插值近似已见过的距离。

**Q：为什么不同注意力头的 RoPE 频率不同？**
> 低频分量捕获长距离依赖，高频分量捕获短距离依赖，不同头分工处理不同尺度的位置关系。
